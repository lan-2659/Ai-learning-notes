onCreate()
    这是 Activity 生命周期中的第一个回调方法。Activity 在首次启动时会调用 onCreate()。
    通常在此方法中进行视图的初始化（setContentView()），并恢复或初始化一些必要的数据。
    在 onCreate() 中，你可以使用 Bundle 对象来恢复之前保存的状态。

onStart()
    在 Activity 对用户可见之前调用，表示 Activity 正在变得可见但还未可交互。
    你可以在此方法中进行一些准备工作，例如启动动画、刷新 UI 等。

onResume()
    该方法在 Activity 将开始与用户交互时被调用。Activity 成为前台并获得焦点时会触发。
    在这里，通常你会恢复一些资源（例如传感器、动画、音频等）并开始接收用户输入。
    
onPause()
    当另一个 Activity 或应用覆盖当前 Activity 时，onPause() 被调用。此时 Activity 不再是用户交互的焦点。
    在此方法中，通常会保存数据、停止动画、释放资源等。

onStop()
    当 Activity 不再对用户可见时调用，意味着 Activity 被另一 Activity 或应用遮挡，或者应用被最小化。
    可以在此方法中释放占用的资源（如数据库连接、文件、网络请求等）。

onRestart()
    当 Activity 从停止状态恢复时调用。通常在 onStop() 后恢复活动并重新显示给用户时调用。
    该方法通常用于恢复 onStop() 时暂停的功能。

onDestroy()
    当 Activity 即将销毁时调用。系统回收 Activity 的资源（包括视图、内存等）时会触发此方法。
    在此方法中，通常会释放所有资源（例如停止线程、释放占用的内存等）。
